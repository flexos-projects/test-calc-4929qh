# docs/

The 8 holy docs. Product specification from vision to content. Flat folder — each doc is a holistic overview of its type; per-item detail lives in `specs/`.

```
000-seed.md        — Optional. Original seed input, never modified after creation.
001-vision.md      — What we're building, for whom, why
002-features.md    — What the product does
003-pages.md       — What screens exist
004-database.md    — What data those pages need
005-flows.md       — How users move through pages
006-design.md      — What it looks and feels like
007-technical.md   — What stack serves this
008-content.md     — What copy, records, and CMS content fill the pages
```

## The dependency chain

```
Vision → Features → Pages → Database → Flows → Design → Technical → Content
```

Each doc depends on all docs before it. Never skip or reorder. When a spec in `specs/` changes significantly, the matching holy doc is regenerated by `flexos-spec-editor`.

The **Library rail** shows holy docs pinned at the top of each category, with matching specs listed below.

See MEGA-SPEC §7 for the full pattern.
